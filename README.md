# lameR_bot

Telegram-бот, разработанный с использованием фреймворка aiogram 3.x и асинхронного взаимодействия с базой данных MySQL через SQLAlchemy.

## Описание

lameR_bot - это Telegram-бот, который использует современные технологии для обеспечения асинхронной работы и масштабируемости. Проект включает:

- Взаимодействие с Telegram API через библиотеку aiogram
- Асинхронное подключение к базе данных MySQL через SQLAlchemy
- Систему миграций базы данных с использованием Alembic
- Структурированную архитектуру с обработчиками команд, клавиатурами, посредниками и утилитами

## Зависимости

Проект использует следующие основные зависимости:

- aiogram (dev-3.x) - фреймворк для взаимодействия с Telegram Bot API
- SQLAlchemy (2.0.19) - ORM для работы с базой данных
- aiomysql (0.2.0) - асинхронный драйвер для MySQL
- alembic (1.11.1) - инструмент для миграций базы данных

Полный список зависимостей можно найти в файле requirements.txt

## Структура проекта

- `main.py` - точка входа в приложение
- `bot/` - основной модуль бота
  - `__main__.py` - запуск бота с настройкой диспетчера и среды
  - `db/` - модели и настройки базы данных
  - `filters/` - фильтры для обработки сообщений
  - `handlers/` - обработчики команд и сообщений
  - `keyboards/` - клавиатуры для взаимодействия с пользователем
  - `middlewares/` - посредники для обработки обновлений
  - `utils/` - вспомогательные утилиты
- `migration/` - настройки и версии миграций Alembic

## Настройка окружения

Для запуска бота необходимо настроить переменные окружения:

- `token_api` - токен для доступа к Telegram Bot API
- `db_user` - имя пользователя для подключения к базе данных
- `db_password` - пароль для подключения к базе данных
- `db_host` - хост для подключения к базе данных
- `database` - имя базы данных

## Запуск

Для запуска бота выполните:

```bash
python main.py
```

## Модель данных

В проекте реализована модель User для хранения информации о пользователях бота:

- `user_id` - уникальный идентификатор пользователя
- `username` - имя пользователя (необязательно)
- `reg_date` - дата регистрации
- `upp_date` - дата последнего обновления

Убедитесь, что у вас установлены все зависимости из файла requirements.txt перед запуском бота.
